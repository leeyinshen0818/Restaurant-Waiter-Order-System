# UI Polish Checklist

Use this checklist before capturing final report screenshots.

## Global checks

- Theme colours match the warm restaurant palette on every screen.
- Cards, buttons, text fields, chips, dialogs, switches, and SnackBars look consistent.
- No text or buttons overflow on a narrow Android emulator.
- Long menu/order names wrap or truncate without breaking controls.
- Keyboard does not hide Save, Update, Review, or Place Order actions.
- Loading buttons disable repeated taps and show progress.
- Empty/error states show an icon, clear text, and useful action where needed.
- Status colours always appear with readable labels, not colour alone.
- Bottom navigation is visible, compact, and does not cover content.
- Dialogs show safe/destructive actions clearly.

## Screenshot-ready screens

- Menu List: multiple items, multiple categories, one unavailable item, edit/delete controls, switch.
- Add/Edit Menu Item: name, price, category, availability, Save/Update button.
- Orders List: summary counts, status filters, several orders with different statuses.
- New Order: selected table, menu choices, item quantities, current order, total.
- Order Detail: table number, lifecycle, order items, total, next action.
